# API Reference

Welcome to the Limitly API reference. This interactive documentation is powered by OpenAPI 3.0.3 and provides real-time testing capabilities.

## Quick Start

### Authentication

All API requests require authentication using your API key:

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.limitly.dev/v1/users
```

### Base URLs

- **Production**: `https://api.limitly.dev`
- **Sandbox**: `https://sandbox-api.limitly.dev`

### Rate Limits

API requests are subject to rate limiting based on your plan:

- **Free Plan**: 100 requests/hour
- **Pro Plan**: 1,000 requests/hour
- **Enterprise Plan**: 10,000 requests/hour

## Response Format

All API responses follow a consistent format:

### Success Response

```json
{
  "data": {
    // Response data
  },
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 100,
    "pages": 5
  }
}
```

### Error Response

```json
{
  "error": "Error message",
  "code": "ERROR_CODE",
  "details": {
    // Additional error details
  }
}
```

## Common HTTP Status Codes

- `200` - Success
- `201` - Created
- `400` - Bad Request
- `401` - Unauthorized
- `404` - Not Found
- `429` - Too Many Requests
- `500` - Internal Server Error

## Examples

### Creating a User

```bash
curl -X POST \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "plan_id": "plan_1234567890abcdef"
  }' \
  https://api.limitly.dev/v1/users
```

### Validating a Request

```bash
curl -X POST \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "api_key": "lk_1234567890abcdef",
    "endpoint": "/api/users",
    "method": "GET",
    "ip_address": "192.168.1.1"
  }' \
  https://api.limitly.dev/v1/validate
```

## Best Practices

1. **Always Handle Errors**: Check for error responses and handle them appropriately
2. **Use Pagination**: For list endpoints, use pagination to handle large datasets
3. **Cache Responses**: Cache responses when appropriate to reduce API calls
4. **Monitor Rate Limits**: Track your rate limit usage to avoid hitting limits
5. **Validate Input**: Use the OpenAPI schema to validate your requests
6. **Use HTTPS**: Always use HTTPS in production environments
7. **Keep API Keys Secure**: Never expose API keys in client-side code
