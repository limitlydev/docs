# API Keys Endpoints

Manage API keys for your users with the Limitly API.

## GET /v1/keys

List all API keys for your account.

### Request

```bash
curl -X GET \
  -H "Authorization: Bearer lk_1234567890abcdef1234567890abcdef" \
  https://api.limitly.com/v1/keys
```

### Query Parameters

<ParamField query="page" type="number">
  Page number for pagination (default: 1)
</ParamField>

<ParamField query="limit" type="number">
  Number of items per page (default: 20, max: 100)
</ParamField>

### Response

```json
{
  "data": [
    {
      "id": "key_1234567890abcdef",
      "name": "Production API Key",
      "api_key": "lk_1234567890abcdef1234567890abcdef",
      "user_id": "user_1234567890abcdef",
      "plan_id": "plan_1234567890abcdef",
      "status": "active",
      "created_at": "2024-01-01T00:00:00Z",
      "last_used": "2024-01-01T12:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 1,
    "pages": 1
  }
}
```

## POST /v1/keys

Create a new API key.

### Request

```bash
curl -X POST \
  -H "Authorization: Bearer lk_1234567890abcdef1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production API Key",
    "user_id": "user_1234567890abcdef",
    "plan_id": "plan_1234567890abcdef"
  }' \
  https://api.limitly.com/v1/keys
```

### Request Body

<ParamField body="name" type="string" required>
  Name for the API key
</ParamField>

<ParamField body="user_id" type="string" required>
  ID of the user who will own this API key
</ParamField>

<ParamField body="plan_id" type="string" required>
  ID of the plan to associate with this API key
</ParamField>

### Response

```json
{
  "data": {
    "id": "key_1234567890abcdef",
    "name": "Production API Key",
    "api_key": "lk_1234567890abcdef1234567890abcdef",
    "user_id": "user_1234567890abcdef",
    "plan_id": "plan_1234567890abcdef",
    "status": "active",
    "created_at": "2024-01-01T00:00:00Z"
  }
}
```

## Error Responses

### 401 Unauthorized

```json
{
  "error": "Invalid API key",
  "code": "INVALID_API_KEY"
}
```

### 404 Not Found

```json
{
  "error": "API key not found",
  "code": "API_KEY_NOT_FOUND"
}
```

### 429 Too Many Requests

```json
{
  "error": "Rate limit exceeded",
  "code": "RATE_LIMIT_EXCEEDED"
}
```

## API Key Object

<ParamField body="id" type="string">
  Unique identifier for the API key
</ParamField>

<ParamField body="name" type="string">
  Human-readable name for the API key
</ParamField>

<ParamField body="api_key" type="string">
  The actual API key value (starts with 'lk_')
</ParamField>

<ParamField body="user_id" type="string">
  ID of the user who owns this API key
</ParamField>

<ParamField body="plan_id" type="string">
  ID of the plan associated with this API key
</ParamField>

<ParamField body="status" type="string">
  Current status of the API key (active, inactive)
</ParamField>

<ParamField body="created_at" type="string">
  ISO 8601 timestamp when the API key was created
</ParamField>

<ParamField body="last_used" type="string">
  ISO 8601 timestamp of the last API request using this key
</ParamField>
