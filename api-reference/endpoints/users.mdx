# Users Endpoints

Manage user accounts with the Limitly API.

## GET /v1/users

List all users for your account.

### Request

```bash
curl -X GET \
  -H "Authorization: Bearer lk_1234567890abcdef1234567890abcdef" \
  https://api.limitly.com/v1/users
```

<ParamField query="page" type="number">
  Page number for pagination (default: 1)
</ParamField>

<ParamField query="limit" type="number">
  Number of items per page (default: 20, max: 100)
</ParamField>

### Response

```json
  }
}
```

## User Management Best Practices

### User Status Management

- **active**: User can make API requests
- **inactive**: User cannot make API requests but account is preserved
- **suspended**: User account is temporarily suspended

### Plan Assignment

When updating a user's plan, consider:

1. **Plan Compatibility**: Ensure the new plan supports the user's current usage
2. **Grace Period**: Consider providing a grace period when downgrading plans
3. **Usage Migration**: Handle existing usage data appropriately

### Security Considerations

1. **Email Validation**: Always validate email addresses
2. **Access Control**: Implement proper access controls for user management
3. **Audit Logging**: Log all user management actions for compliance
4. **Data Privacy**: Ensure compliance with data protection regulations
