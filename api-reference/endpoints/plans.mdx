# Plans Endpoints

Manage usage plans for your users with the Limitly API.

## GET /v1/plans

List all plans for your account.

### Request

```bash
curl -X GET \
  -H "Authorization: Bearer lk_1234567890abcdef1234567890abcdef" \
  https://api.limitly.com/v1/plans
```

### Query Parameters

<ParamField query="page" type="number">
  Page number for pagination (default: 1)
</ParamField>

<ParamField query="limit" type="number">
  Number of items per page (default: 20, max: 100)
</ParamField>

### Response

```json
{
  "data": [
    {
      "id": "plan_1234567890abcdef",
      "name": "Basic Plan",
      "description": "Plan for basic users",
      "max_requests": 10000,
      "request_period": "month",
      "features": ["api_access", "basic_analytics"],
      "status": "active",
      "created_at": "2024-01-01T00:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 1,
    "pages": 1
  }
}
```

## POST /v1/plans

Create a new plan.

### Request

```bash
curl -X POST \
  -H "Authorization: Bearer lk_1234567890abcdef1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Basic Plan",
    "description": "Plan for basic users",
    "max_requests": 10000,
    "request_period": "month",
    "features": ["api_access", "basic_analytics"]
  }' \
  https://api.limitly.com/v1/plans
```

### Request Body

<ParamField body="name" type="string" required>

</ParamField>

<ParamField body="description" type="string">
  Description of the plan
</ParamField>

<ParamField body="max_requests" type="number" required>

</ParamField>

<ParamField body="request_period" type="string" required>
  Time period for the limit (day, week, month, year)
</ParamField>

<ParamField body="features" type="array">
  Array of features included in this plan

### Response

```json
    "id": "plan_1234567890abcdef",
    "name": "Basic Plan",
    "description": "Plan for basic users",
    "max_requests": 10000,
    "request_period": "month",

```

  Unique identifier for the plan

## Supported Plan Periods

- **day**: 24-hour period
- **week**: 7-day period
- **month**: 30-day period
- **year**: 365-day period

## Common Plan Features

- **api_access**: Basic API access
- **basic_analytics**: Basic usage analytics
- **advanced_analytics**: Advanced analytics and reporting
- **priority_support**: Priority customer support
- **custom_limits**: Custom rate limiting rules
- **webhook_notifications**: Webhook notifications for events
